Open-source local-first context compiler

GotSaeng OS

흩어진 Markdown 메모, 결정, 할 일, 질문, 리서치 기록을 로컬에서 읽고 사람과 AI가 바로 이어받을 수 있는 구조화된 context pack으로 컴파일하는 오픈소스 프로젝트입니다.

비기술 소개 기술 소개 산출물
Local-first 데이터를 외부 서비스로 보내지 않고 사용자의 컴퓨터 안에서 처리합니다.
Markdown-native Obsidian 같은 Markdown 지식 저장소를 중심으로 동작합니다.
Deterministic LLM 추측이 아니라 명시적 규칙과 스키마로 안정적인 결과를 만듭니다.
Model-ready 사람이 읽는 문서와 자동화에 쓰는 JSON을 함께 생성합니다.

코딩을 모르는 사람에게

GotSaeng OS는 긴 프로젝트를 다시 시작할 때 필요한 맥락을 되찾아주는 정리 도구입니다. 예전 메모를 하나씩 뒤지지 않아도 현재 상황, 중요한 결정, 남은 할 일, 위험 요소, 오래된 정보를 한눈에 볼 수 있게 만듭니다.

문제

프로젝트 맥락은 쉽게 흩어집니다

회의록, 아이디어, 채팅 기록, TODO, 리서치, 결정 사항이 서로 다른 파일과 날짜에 흩어지면 프로젝트가 멈춘 뒤 다시 이어가기 어렵습니다.

해결

흩어진 기록을 context pack으로 묶습니다

GotSaeng OS는 로컬 메모를 읽고 핵심 정보를 종류별로 정리해 사람과 AI가 바로 이해할 수 있는 프로젝트 설명서 묶음으로 만들어줍니다.

주요 기능

메모 스캔

로컬 Markdown vault를 훑어보고 프로젝트에 필요한 노트만 대상으로 삼습니다. Obsidian 설정, 빌드 결과물, 이전 생성물 같은 폴더는 제외합니다.

중요 정보 추출

사실, 결정, 할 일, 위험, 가정, 질문, 인사이트를 구분해 뽑습니다. 한국어와 영어 마커를 모두 지원합니다.

오래된 맥락 감지

업데이트 날짜와 열린 작업을 기준으로 다시 확인해야 할 항목을 표시합니다.

출처 품질 점수

제목, 태그, 업데이트 날짜, 노트 유형 같은 로컬 메타데이터를 바탕으로 각 항목의 출처 관리 상태를 strong, moderate, weak로 나눕니다.

변화 추적

이전 compile 결과와 현재 결과를 비교해 새로 생긴 항목, 바뀐 항목, 새로 오래된 항목, 해결된 항목을 보여줍니다.

Obsidian 연결

Obsidian 안에서 context pack 생성, 주간 리뷰, LLM handoff, vault 검증, Report Hub 열기를 실행할 수 있습니다.

기술 지식이 있는 사람에게

GotSaeng OS는 TypeScript monorepo로 구성된 deterministic context compiler입니다. 현재 버전은 LLM API, RAG, vector database, cloud sync 없이 로컬 파일과 명시적 규칙만으로 context artifact를 생성합니다.

파일 탐색

fast-glob로 vault 파일을 스캔하고 ignore rule을 적용합니다. 결과를 정렬해 같은 입력에 대해 안정적인 처리 순서를 보장합니다.

Markdown 파싱

gray-matter로 YAML frontmatter와 Markdown body를 분리합니다. title, tags, created, updated, type 메타데이터를 정규화합니다.

스키마 계약

zod로 ContextPack, ExtractedItem, CompileReport, ContextManifest, MemoryDiff 스키마를 런타임 검증합니다. TypeScript 타입은 스키마에서 파생합니다.

정보 추출

정규식과 Markdown heading/section 구조를 사용합니다. 명시적 마커, task list, 섹션명 기반 추론을 조합해 fact, decision, action, risk, assumption, question, insight를 추출합니다.

결정론적 ID

source path, item kind, normalized text 기반으로 deterministic ID를 만듭니다. manifest diff와 반복 가능한 출력 생성의 기반입니다.

Stale detection

updated 날짜와 open/active action 상태를 기준으로 stale 항목을 계산합니다. date provider를 주입할 수 있어 테스트에서 시간을 고정할 수 있습니다.

Provenance scoring

의미론적 사실 검증이 아니라 metadata heuristic입니다. updated, created, title, tags, note type, status, priority signal을 점수화해 weak, moderate, strong으로 분류합니다.

CLI와 adapter 분리

core compiler는 packages/core에 두고, commander 기반 CLI는 packages/cli, Obsidian shell은 apps/obsidian-plugin에 둡니다.

생성되는 산출물

결과물은 사람이 읽는 Markdown 문서와 도구가 읽는 JSON 문서로 나뉩니다. Markdown은 프로젝트 리뷰와 AI handoff에 적합하고, JSON은 자동화와 변화 추적에 적합합니다.

PROJECT_CONTEXT.md 현재 목표, 핵심 사실, 결정, 액션, 위험, 질문을 모은 프로젝트 요약입니다.
MEMORY_SNAPSHOT.md 오래 보존할 사실, 인사이트, 가정, 최근 업데이트와 경고를 정리합니다.
DECISION_LOG.md 프로젝트에서 내려진 주요 결정을 출처별로 정리합니다.
ACTION_BACKLOG.md open, active, unknown, done 상태별 작업 목록을 만듭니다.
RISK_REGISTER.md 위험 요소와 불확실성을 따로 모아 검토할 수 있게 합니다.
OPEN_QUESTIONS.md 아직 답이 필요한 질문을 정리합니다.
STALE_CONTEXT.md 오래되었거나 다시 확인해야 할 맥락을 표시합니다.
SOURCE_PROVENANCE.md 출처 품질 점수와 경고를 보여줍니다.
MEMORY_DIFF.md 이전 compile과 현재 compile 사이의 변화를 비교합니다.

현재 하지 않는 것

이 프로젝트의 현재 방향은 작고 검증 가능한 local-first compiler입니다. 다음 기능들은 현재 버전의 목표가 아닙니다.

SaaS, cloud sync, authentication, payments, mobile app은 포함하지 않습니다.

OpenAI, Anthropic, Gemini SDK나 LLM API 호출을 포함하지 않습니다.

Vector database, RAG, autonomous research agent는 현재 기능이 아닙니다.

Provenance scoring은 의미론적 사실 검증이 아니라 metadata triage입니다.

GotSaeng OS 生 흩어진 생을 다시 손에 쥐다.
